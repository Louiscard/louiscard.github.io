Ulysses 访谈：谈谈写作、Ulysses 和我的利器

发表于 2019-05-06 | 分类于 其他 | 阅读次数

1. 跟我们说说你吧，你现在做什么呢？

我的主要工作是在一家科技公司负责用户增长和客户成功，我们通过大数据和人工智能技术研发辅助法官、检察官、律师更高效地工作的工具，解放法律人的生产力。

我的斜杠事业是为知识工作者提供高效生产力系统的咨询，周末不加班的时候会通过在行接单，在咖啡馆或者 WeWork 给大家做分享。除此之外，我还是少数派的签约作者，最近刚刚完成了 Evernote 教程的更新，最近又开启了 OmniOutliner 教程的编写。

2. 写作在你生活中扮演怎样的角色呢？你通常的写作主题是什么？

写作在我的生活中扮演着非常多样的角色，每一种角色都各不相同，但对我而言，每一个都非常重要：

知识管理的闭环：10年之前，我用 wiki 工具打造了第一代知识管理系统，经过无数次崩溃重启、迭代优化，这套系统终于齿轮咬合，运行平稳，基本形成了一整套以 Evernote、OmniOutliner、XMind、Anki、Ulysses 等工具为基础，信息的输入、整理、内化、输出为流程的个人知识管理解决方案。一开始，我着迷于大吞吐量的输入、深度高频的思考，而如今，我更沉醉于对某个主题持续稳定地输出。知识管理本身更趋近于手段，而写作，大大增强了它的目的性，让信息的流转形成真正的闭环。

品牌打造的手段：我觉得写作是建立个人品牌最有效的方式，几乎没有之一。相对于线下咨询和企业培训，写作的杠杆要高出几个数量级。所以，相对于贩卖时间，我越来越倾向于写成文章，有价值的内容会以更低的成本被更多人看到，而这些内容聚集起来还会成为咨询方案和培训产品的基础材料。有时候拜访客户，对方收起我的名片，笑着说：我看过你写的文章，每到这时候，我就知道，这一次交流的沟通成本和信任成本，都会被降到最低，大概率我会有一段相对美好的体验。

自我认知的途径：我非常重视总结复盘，用 Day One 写了好几年的日记。写作是一件边界很大的事情，我从来不觉得只有发表在公众号和博客的内容才算正经的写作，恰恰相反，我觉得那些可能永远不会公之于众的文字对自己反而更重要。每天的任何时间，任何场景，我都可能掏出手机或者打开电脑，噼里啪啦敲几行，时时刻刻的总结复盘。在睡觉之前，也会回顾自己一整天的生活，把它们记录在日记中。写下来的过程，也是自我治愈的过程，这样的写作帮我度过最艰难的日子，让我能够在更多时间保持内心的喜乐平和。

3. 你最喜欢 Ulysses 的是什么？

Ulysses 有非常多优秀的功能，比如打字机模式、黑暗主题、灵活的快捷键设置、支持 Text Expander 等等，然而我最喜欢 Ulysses 的一个原因是：Ulysses 大幅度减少了我的写作成本。

比如对多个设备的支持和同步机制，为我提供了一个可以随时随地进入写作状态的绝佳体验，我可以在 MacBook、iPad、iPhone 上无缝接续自己的写作活动。因此，吃完饭后的 10 分钟，会议开始前的 10 分钟，地铁通勤的半小时，都可以变成高质量的写作时间。一旦「开始写作」的成本被大幅度减少，只要你开始敲键盘，剩下的问题，都不构成实质性威胁。

内置的文库功能也是我选择 Ulysses 的重要原因。一直以来，我的写作模式更倾向于「计划经济」，我会走完梳理思路、收集资料、修改提纲、完成初稿、修改校对的全部流程。整个过程需要多个工具配合，因此，大部分时候也只能在 MacBook 完成。直到我开始使用 Ulysses 才彻底改变了这个习惯，Ulysses 提供的内置文库轻松管理着我所有的写作项目，我的全部选题和卡片都在 Ulysses 存储。

我模仿了一个 Scrivener 的写作项目设置，借鉴了纳博科夫的卡片写作模式，把所有的主题分为 Drafts、Cards、Archive 三个组，在 Drafts 里完成当下的文章编写，在 Cards 积累日常各种这个主题的零碎思考，而所有发表的文章会移动到 Archive 中归档。所有的写作项目都可以在一个工具上完成，且可以随时开始，说实话，这对我的帮助太大了！

最后还有它的搜索功能，Ulysses 虽然是一款写作工具，但是它的搜索功能做得相当出色，这也我能把它作为核心生产力工具的重要原因之一。比如，突然想明白一个道理，觉得简直是妙手偶得，我一般会先在 Ulysses 写一张卡片，但有一个不可回避的问题，这个主题会不会在之前已经写过了呢？这时候我就会在这个主题的 Cards 组中先查询下关键词。还有时候需要写一篇文章，除了搜索网络、Evernote 之外，自己之前写过的文章也是重要的资料来源，这时候全库搜索就显得非常好用啦。

4. 除了 Ulysses 你还会用什么其他写作工具呢？或者有什么你特别喜欢的 App 想与我们分享？

GitHub：与编辑的协同利器。在写 Evernote 付费教程的时候，我和编辑全程通过 GitHub 进行协作和版本控制，我们在 GitHub 创建一个项目，使用本地客户端管理文档，用 Ulysses 添加外部文件夹，这样在 Ulysses 写作，一键上传到 GitHub，每一次修改的版本都会保存，编辑改完之后，我这边能清楚地看到所有修改的地方，即便 3 个人一起也不会混乱，当然 Ulysses 自己是有版本快照功能功能的，如果不涉及到协作，自带的功能就完全够用。

Markdown here： 一秒钟排版。这是一款我非常喜欢的 Chrome 插件，最开始是帮助用 Markdown 写邮箱的人渲染出漂亮的格式，我们可以用它给微信公众号的文章排版，真的只需要 1 秒钟，大幅度减少发布公众号文章的后期成本。

Trello：管理写作选题。想写的东西太多，不知道该选哪个，或者想写的东西太少，总是无从下笔。这个困扰了我很久，也尝试过用很多方法尝试管理我的写作选题，但效果都一般。去年年底，我看了少数派的 Trello 选题看板，这给了我巨大的启发。

于是我自己也马上制作了属于自己的写作选题看板，维护我想要写的主题。

5. 你理想的生活或是工作状态是怎样的呢？

我希望有越来越强大的掌控力和节奏感（大概是因为天蝎座），每天在 6 点之前起床，完成自己的晨间仪式，跑步，清空收件箱，给家人做早餐，然后安排好一天的事项，全身心投入到工作中、下班回家可以看看电影，读读小说，写 1000 字，完成睡前仪式，获得 7 小时以上的安眠。关键是，能够始终做自己喜欢的事情，享受过程。

6. 未来你对 Ulysses 有什么期待？或是希望增添的功能？

作为 Ulysses 的铁粉，在这个环节就不客气了，一口气提三个需求：

1）支持发布到微信公众号：可以像发布到 Medium 一样把文章直接发布到微信公众号，如果能带着我喜欢的样式那就太完美了。

2）生成长图片：写作的内容无论是 Markdown 格式还是 Html 格式，可以直接导出成图片，很多时候，在写成文章之前，我希望以图片的形式发布在微信朋友圈或者微博上。

3）内置图床功能：图片粘贴到 Ulysses 之后，可以一键将文章中的图片上传到图床服务器并补充好地址，不再需要第三方工具，对每一张图片逐一上传获取地址再贴到图片。

7. 最后送给 Ulysses 一句话吧。

希望 Ulysses 越来越好，为更多创造者提供美好的写作体验。
